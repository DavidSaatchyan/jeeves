# T3.3 — Error Recovery Protocol

**Status:** Done
**Date:** 2026-06-10
**Related:** All agents, T0.1 Plugin Architecture

## Principles

1. **Never crash the session** — agent errors are caught and reported
2. **Retry once automatically** — transient failures get one retry
3. **Fail gracefully** — log the error, inform the user, suggest alternative
4. **Store failure patterns** — learn from failures to avoid repetition

## Error Categories

| Code | Description | Action |
|------|-------------|--------|
| `E_TIMEOUT` | Task took too long (>2min) | Retry with simpler prompt |
| `E_TOOL_FAIL` | Tool execution failed | Retry once, then skip |
| `E_PARSE` | Agent output unparseable | Regenerate output |
| `E_PERMISSION` | Agent lacks permission | Request approval, skip if denied |
| `E_CONTEXT` | Task context too large | Split into sub-tasks |
| `E_MCP` | MCP server unreachable | Fall back to no-memory mode |
| `E_UNKNOWN` | Unexpected error | Report to user, log full trace |

## Recovery Workflow

```
Error detected
  ├─ E_TIMEOUT     → swarm retries with shorter prompt
  ├─ E_TOOL_FAIL   → retry once → skip tool → continue
  ├─ E_PARSE       → ask agent to regenerate
  ├─ E_PERMISSION  → prompt user → skip if denied
  ├─ E_CONTEXT     → split into parallel sub-tasks
  ├─ E_MCP         → disable memory, continue without persistence
  └─ E_UNKNOWN     → log full trace → inform user → abort subtask
```

## Fallback Chain

When a sub-agent fails:

1. **Retry** with same agent, same prompt (if E_TIMEOUT/E_TOOL_FAIL)
2. **Fallback agent** — use simpler agent (e.g., researcher instead of architect)
3. **Manual** — present options to user, let them decide
4. **Skip** — mark task as failed, continue pipeline

## File-Based Error Log

Each error is logged to `docs/errors/<task-id>.json`:

```json
{
  "timestamp": "2026-06-10T16:00:00Z",
  "task": "T2.2-devops-agent",
  "agent": "implementer",
  "error_code": "E_TOOL_FAIL",
  "error": "Edit tool failed: file not found",
  "recovery": "retry_once",
  "recovery_result": "success"
}
```

## Integration

- Plugin `session.error` hook routes errors here
- Each agent's `.md` file has an Error Recovery section
- swarm agent implements the retry logic
- MCP stores failure patterns for future avoidance

## Anti-Patterns

- ❌ Silent failure — catching errors without logging
- ❌ Infinite retry loops — max 1 retry per task
- ❌ Uncaught exceptions in plugin hooks
- ❌ Blocking indefinitely on user input
