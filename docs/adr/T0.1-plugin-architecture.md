# T0.1 — Plugin Architecture Doc

**Status:** Done
**Date:** 2026-06-10
**Related:** T0.2 Plugin Refactor, T3.1 MCP Server

## Current State

`opencode/plugins/openflo-core.ts` is a minimal plugin (~19 lines) with a single `tool.execute.after` hook that logs to console. No proper structure, no error handling, no lifecycle.

## Requirements

OpenFlo plugins need to:
1. Intercept tool calls for memory operations (learn/recall/forget)
2. Handle error recovery (auto-retry, logging)
3. Provide lifecycle hooks (init, before task, after task)
4. Be testable and maintainable

## Architecture

### Plugin Types

```
plugins/
├── core/           # openflo-core: lifecycle, error recovery, memory hooks
├── memory/         # (future) MCP client integration
├── monitoring/     # (future) telemetry, metrics
└── federation/     # (future) cross-project sync
```

### Core Plugin Structure

```typescript
plugin({
  name: 'openflo-core',
  version: '0.1.0',
  hooks: {
    config(config) { /* validate OpenFlo config */ },
    'tool.execute.before'({ tool, args }) { /* check memory preconditions */ },
    'tool.execute.after'({ tool, args, result }) { /* auto-learn patterns */ },
    'session.initialize.after'({ session }) { /* bootstrap memory connection */ },
    'session.error'({ error }) { /* error recovery dispatch */ },
  }
})
```

### Hook Responsibilities

| Hook | Purpose |
|------|---------|
| `config` | Validate opencode.json has OpenFlo agents, MCP, skills |
| `tool.execute.before` | Check if we have context in MCP for this tool call |
| `tool.execute.after` | Auto-store successful patterns, detect failures for recovery |
| `session.initialize.after` | Warm up MCP connection |
| `session.error` | Route errors to recovery protocol |

### Error Handling

```typescript
class PluginError extends Error {
  constructor(
    message: string,
    public code: string,
    public recoverable: boolean,
    public retryCount?: number
  ) { super(message); }
}
```

All hooks wrap in try/catch — plugin errors never crash the session.

### Testing Strategy

- Unit tests per hook with mock context
- Integration test: run OpenCode session with plugin loaded
- Test error recovery: inject failures, verify fallback

## Constraints

- Plugin cannot block more than 5s total (non-blocking design)
- No external dependencies beyond what OpenCode provides
- Must be compatible with OpenCode v1.17+
- All MCP communication goes through the main agent's MCP servers
