**Status:** Done

## Agent Model

OpenFlo agents are **OpenCode `.md` files** with YAML frontmatter. They are NOT services — they are named prompts with structured instructions.

### Frontmatter Schema

```yaml
---
description: What this agent does (1-2 sentences)
mode: plan | build         # OpenCode mode
model:
  - fast: haiku-4.5       # Model for simple tasks
  - smart: sonnet-4.6     # Model for complex reasoning
permission:
  - read: allow
  - edit: allow | ask     # Can this agent edit files?
  - bash: allow | ask     # Can this agent run shell?
  - audit: allow          # Reviewer/security only
color: hex-color           # Visual identifier in swarm output
---
```

### Body Structure

Every agent must have these sections:

1. **Process** — 3-5 step workflow (numbered)
2. **Output Format** — structured return data
3. **Error Recovery** — what to do on failure
4. **Anti-Patterns** — what NOT to do

### Agent Categories

| Category | Agents | Model |
|----------|--------|-------|
| **Core** | swarm, implementer, architect, reviewer, tester, security | Smart |
| **Support** | debugger, refactorer, documenter, researcher | Smart/Fast |
| **Expansion** | devops, perf, migrator, deps, i18n, legal, ux | Fast |

### Communication Model

```
User → swarm (orchestrator)
  → architect (plan)
  → implementer (code)
  → reviewer (audit)
  → tester (verify)
  → security (check)
  → swarm (report back to user)
```

Each transition passes through `task` with context. No direct inter-agent communication.

### Context Passing Standard

When delegating, use this format:

```
Context: <prior decisions, architecture, constraints>
Goal: <what to produce>
Files: <paths to read or modify>
Constraints: <security, perf, style requirements>
Return: <expected format>
```

### Quality Gates

Every task before completion:
- [ ] Compiles / lints
- [ ] Tests pass
- [ ] No CRITICAL security findings
- [ ] Key decisions stored in memory (openflo_learn)
- [ ] User informed

## Expansion Agent Template

New agents should follow this template (80-150+ lines):

```markdown
---
description: ...
mode: plan|build
model:
  - fast: ...
  - smart: ...
permission:
  - read: allow
  - edit: allow|ask
  - bash: allow|ask

---

<Process: 3-5 steps>

<Output Format>

<Error Recovery>

<Anti-Patterns: 3-5 items>
```
