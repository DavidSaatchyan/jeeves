Proactive engine — optional

What is this?

Normally the agent only replies when a user writes first. Proactive engine flips that: Jeeves writes to the user first when something looks wrong — for example, when a user's activity suddenly drops.

Typical use: a user stops logging in, agent pings them with "Hi! I noticed you haven't been around. Need any help?" — recovers churn before it happens.

How to set it up

1
Provide a metric URL from your product/CRM that returns a numeric activity time-series for a user (e.g. logins per day for the last week). Use {id} for the user ID.
2
Pick a drop threshold in percent. If today's value is X% below the 3-day average, Jeeves writes to the user.
3
That's it. The worker checks every hour and won't bother the same user more often than every 3 days.

Expected response shape (any of these works): {"data":{"series":[10,12,11,2]}} · {"series":[10,12,11,2]} · {"data":{"activity":[10,12,11,2]}}. The last value is "today", the rest are the baseline.
