# План перехода на новую структуру UI

## Оценка готовности

| Раздел | Готовность | Что есть | Что нужно |
|--------|-----------|----------|-----------|
| 1. Дашборд / Аналитика | 0% | Нет ничего | Весь раздел с нуля |
| 2. Inbox (Диалоги) | ~40% | Список диалогов, чат, API | Трёхпанельный layout, handover, Copilot, контекст CRM |
| 3. Агенты (Команда ИИ) | 0% UI / ~60% backend | agents/ пакет (incoming_line, registry) | Список агентов, настройки (4 таба), песочница |
| 4. База знаний (RAG) | ~50% | Загрузка/список файлов, базовый Q&A | Дерево папок, редактор, chunking, RAG simulator, синхрон цен |
| 5. Интеграции | ~50% | Cliniko, Pabau, WhatsApp, Widget | Integration Hub, Instagram, QR WhatsApp, статус скрипта |
| 6. Настройки компании | ~30% | API keys, compliance, базовый account | Команда/роли, usage bars, биллинг, AI Activity Logs |

## Phase 0: Общая перекомпоновка (Sidebar + навигация)

**Цель:** Привести сайдбар и базовую навигацию в соответствие с новой структурой, не ломая существующие страницы.

### Что делаем:

1. **Новый сайдбар** в `templates/base.html`:
   ```
   📊 Дашборд
   📥 Inbox
   🤖 Агенты
   📂 База знаний
   🔌 Интеграции
   ⚙️ Настройки компании
   ```

2. **Удалить из сайдбара** (мёртвые ссылки):
   - Секцию «Assistants» (Marketing Funnel, Patient Follow-up)
   - Campaigns
   - Badge-счётчик (стучится в мёртвые API)

3. **Редиректы / заглушки:**
   - `/admin/dashboard` → заглушка (раздел пока пуст)
   - `/admin/agents` → заглушка (будет список агентов)

4. **Сохранить** рабочие страницы как есть:
   - Inbox (`/admin/inbox`) — работает, не трогаем
   - Knowledge (`/admin/knowledge`) — работает
   - Cliniko (`/admin/pabau`, `/admin/cliniko`) — работают, позже переедут в Integrations
   - Settings / Account / Compliance — работают, позже переедут в Settings

5. **Файлы:** `templates/base.html`, `admin/pages.py` (добавить роуты-заглушки)

## Phase 1: Агенты (Agents)

**Почему сначала:** Это ядро продукта, и для него уже есть backend (agents/ пакет). UI с нуля, но scope понятен.

### Подфазы:
1.1. **Список агентов** — страница-сетка карточек (сейчас только Incoming Line)
1.2. **Страница настройки агента** — 4 таба:
   - Личность и характер (имя, ToV, промпт)
   - Навыки и бизнес-правила (capabilities, hard rules)
   - Подключение Базы знаний (выбор папок)
   - Каналы связи (WhatsApp, Widget)
1.3. **Песочница (Playground)** — симулятор чата справа

## Принципы

1. **Сначала Phase 0** — сайдбар и навигация, ничего не ломать
2. **Каждый Phase — отдельный PR/commit**
3. **После Phase 0 можно деплоить** — все старые страницы продолжают работать
4. **Phase 1 (Агенты) — приоритет** — backend готов, UI — самое важное для пользователя
