ТЗ  на  продукт:  Universal  AI  Agent  (UAA)  
Версия  1.0  —  концепция  
 
1.  Ценности  продукта  
№  Проблема  рынка  Наше  решение  
1  Интеграция  с  легаси  CRM  занимает  месяцы  
Self-serve:  клиент  даёт  API- ссылку  →  агент  сам  тестирует  и  подключается  за  5  минут  
2  Данные  не  AI-ready  (разрознены,  грязные)  
Инструмент  «чистки  и  индексации»:  загрузил  PDF/ ссылки  →  готово  
3  Нет  проактивности  —  всё  реактивно  
Детектор  аномалий  +  планировщик  →  агент  пишет  первым  
4  Непонятная  цена  (request  a  quote)  
Pay-per-resolution  +  лимиты  в  личном  кабинете  
5  Enterprise  не  могут  отправлять  данные  в  OpenAI  
Model-agnostic  +  возможность  локального  деплоя  
6  Чёрный  ящик  —  непонятно,  почему  ответил  так  
Логи  решений  +  аудит

5.  Детальное  ТЗ  на  MVP  
5.1.  Что  входит  в  MVP  
Цель  MVP:  клиент  может  за  15  минут  настроить  агента,  который  отвечает  на  вопросы  
из
 
его
 
документов
 
и
 
выполняет
 
1
 
действие
 
(чтение/запись
 
в
 
CRM).
 
Включено  в  MVP  
Модуль  Что  именно  
Knowledge  Manager  Загрузка  .txt,  .pdf,  .md  →  автоматическая  индексация  (chunk  512,  overlap  64)  →  поиск  через  RAG  
CRM  Connector  Подключение  через  REST  API  (клиент  даёт  URL  +  метод  +  маппинг  полей)  →  агент  умеет  читать  и  записывать  2  поля  (например,  tariff,  status)  
Channel  Manager  Веб-виджет  (iframe/JS)  +  REST  API  для  внешних  вызовов  
Agent  Core  Оркестратор  (LangChain),  RAG,  память  диалога  (на  сессию),  1  action  (update  tariff)  
Admin  Dashboard  Регистрация/логин,  загрузка  файлов,  настройка  CRM- подключения,  просмотр  логов  диалогов  
Проактивность  1  сценарий:  падение  метрики  из  CRM  →  агент  пишет  «Нужна  помощь?»  
Не  входит  в  MVP  (будущие  этапы)  ●  WhatsApp  /  Telegram  /  Slack  каналы  ●  Более  1  действия  ●  Проактивные  сценарии  кроме  P1  ●  On-premise  деплой  ●  Сложные  разрешения  RBAC  ●  Интеграция  с  Zapier  /  Make
